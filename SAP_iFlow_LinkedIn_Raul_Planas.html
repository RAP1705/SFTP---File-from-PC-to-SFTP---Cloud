Recently, someone posed a challenge to build an end-to-end file delivery system utilising third party tools and SAP Integration Suite, trial account. I built it and I want to break down exactly how it works. The goal: automate secure file transfer from SAP to a Linux server, with zero manual intervention. Here's the full stack I implemented: 🔹 SAP Integration Suite (iFlow) → Designed the integration flow with a Content Modifier to shape the message payload → Configured the SFTP Receiver Adapter for outbound file delivery 🔹 Network layer → SSH-encrypted connection over the public internet → Fixed egress IP allowlisted on the target server firewall 🔹 Linux VM (Google Cloud) → SFTP daemon (sshd) authenticates the incoming connection → File lands in /home/sftpuser — fully automated, no manual steps The result: a PDF file delivered reliably from SAP to a remote server via a secure, production-grade pipeline. This project gave me hands-on experience across SAP BTP, SFTP protocol configuration, Linux server administration, and cloud networking — end to end. Open to SAP Integration / BTP roles.
